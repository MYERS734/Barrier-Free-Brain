# バリアフリーブレイン（Barrier-Free Brain）

## 🧠 これは何？

発達障害（ADHD/ASD）の当事者である開発者が、自身の困りごとから設計した**「認知負荷が低く、ミスしにくい」自分専用のAI型家計簿・会計システム**のプロトタイプです。

市販の家計簿アプリや会計ソフトにありがちな、
- 情報が詰め込まれすぎてパニックを誘発するUI
- 操作や判断を急がせる複雑な入力構造
- 1つのミスで最初からやり直しになる不親切な設計
といった課題を解決し、**「自分の体の一部のように扱えるツール」**を目指して構築しました。

---

## 🎯 コンセプト＆特性配慮設計

- **焦らせないUI（1ステップ・1タスク）**：Streamlitを活用し、1画面につき「金額確認」「カテゴリ確認」など1つの操作だけに集中できるステップ式画面を設計。
- **認知負荷の徹底的な排除**：画面上の文字やボタンを大きく押しやすく配置。勘定科目は「食費（毎日のごはん・おやつ）」など、直感的に分かりやすい優しい表現を徹底。
- **実用的なOCR×AIデータ修復**：スマホで撮影したレシート画像（iPhoneのHEIC形式に対応）からTesseract OCRで文字を抽出。文字化け（ノイズ）が発生しても、システム内部のAIフィルター（プロンプト設計）によって正しい商品名や金額の整合性をトリプルチェックで推論・自動修復します。

---

## 🛠 技術スタックと選定理由（コスト・セキュリティ最適化）

- **フロントエンド**: Python (Streamlit) / 1タスク集中型のUIを高速にプロトタイピングするため選定。
- **画像・OCR処理**: Pillow / pillow-heif (iPhoneのHEIC形式対応) / pytesseract (Tesseract OCR)
- **バックエンド・データ構造**: Python / 画面は優しい家計簿表現にしつつ、内部データ（JSON）は**簿記2級の複式簿記ロジック**（日付・金額・借方・貸方のオブジェクト構造）でクリーンに生成。将来的には**freee-mcp**を通じて、個人の家計簿から企業の仕訳システムへとシームレスにデータ連携できる拡張性を持たせています。
- **コスト最適化戦略**: ポートフォリオとしての永続的な運用・公開を考慮し、高コストな常時API課金を避け、開発環境でのテストや採用担当者の閲覧時にコストとサーバー負荷をゼロにする「セーフティ・モックデータ連携構造」を採用。

---

## 💼 簿記2級の知識とドメイン応用

本システムは、開発者が持つ**日商簿記2級の知識**をシステム設計に落とし込んでいます。
単にAIに分類を丸投げするのではなく、AIの初期提案（例：夜食としての福利厚生費など）に対し、ユーザーが「休日の私生活の支出であるため、個人用の食費（事業主貸）が正しい」と直感的に手動修正できるコントロールパネルを実装しています。

## 💡 きっかけと背景

就労移行支援施設での訓練中、「入力ミスが多い」「データ処理のスピードが遅い」という指摘を受け、「パソコンを使う仕事は向いていないのではないか」と言われたことが開発のきっかけです。

しかし私は、**「人間の能力が劣っているのではなく、ツールが人間の特性に合っていないだけではないか」**と考えました。
「合わないなら、自分に合う道具を自分で作ればいい」。
このツールを通じて、**「適切な道具（バリアフリーUI）さえあれば、私たちは正確に、圧倒的な成果を出せる」**ということを証明します。
